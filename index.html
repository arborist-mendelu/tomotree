TomoTree

  • Open source software for acoustic tomography of trees.
  • Also allows processing of resistograph data.
  • Combination of tomograph and resistograph data.
  • Easy to use graphical user interface based on Streamlit.
  • Available as Python package or Docker container.

Acoustic rays visualization

  • TomoTree simplifies visualization of acoustic rays in the tree cross-section.
  • Put rays and cross section photo (if available) together.
  • Speed filter for rays allows to focus on the slowest or the fastest rays.
  • Helps to identify problematic rays and improve tomogram quality.

Tomogram reconstructions

  • Several reconstruction methods available.
  • No black box. Each reconstruction method well documented.
  • The tomogram can be compared with the photo of the actual cross section, if available.
  • Colormap for speed visualization can be adjusted.

Speed calibration

  • Speed calibration allows to set the sound speed in the sound wood. This can be used as a reference for tomogram interpretation.
  • The sound speed can be set based on the tangential rays in tomogram.
  • The user can finetune the sound speed by excluding the rays close to the defect.

Binary tomogram

  • Binary tomogram allows to separate the sound wood from the degraded wood or the cavities.
  • The threshold can be set interactively and the effect on the binary tomogram is shown immediately.
  • The area of the sound wood can be calculated and used for further analysis.
  • Given an axis of rotation, the physical characteristics important for tree stability can be estimated.

Getting started

  • The stable GUI interface contains well tested code which allows to utilize TomoTree without programming skills.
  • The development version of the GUI interface contains new features which will be propagated to the stable branch after testing period.
  • You can also run TomoTree locally using Docker or Conda. See the documentation on GitHub.
